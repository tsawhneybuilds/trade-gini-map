Definitions

  • Active Product Gini: concentration across observed positive HS6 product totals within a country-year-flow; absent or zero-trade products are outside the Gini universe.
  • Active Partner Gini: concentration across observed positive destination or source partner totals within a country-year-flow; absent or zero-trade partners are outside the Gini universe.
  • Active Product-partner cell Gini: concentration across observed positive HS6 product-by-partner cells within a country-year-flow; this is not a zero-inclusive universe measure.
  • World-Relative Product Gini: export-only concentration across s_cpt / w_-c,pt, where country harmonized HS6 family shares are compared with leave-one-out world product-family weights in the balanced 2000-2024 sample.
  • Within-product supplier HHI: concentration across supplier countries inside one importer-year-HS6 product.
  • Partner-Gini counterfactual: recomputes Partner Gini after each identified HS6 product is split equally across its observed suppliers; HS6 999999 stays in partner totals but is not treated as a product.
  • Full-diffusion upper bound: recomputes Partner Gini after each HS6 product is split equally across every active partner in the importer-year.
  • Lumpy-product exclusions: HS27 oil/mineral fuels, HS71 precious stones/metals, HS88 aircraft, HS89 ships, and HS93 arms.
  • HS2-preserving benchmark: randomizes within broad HS2 sectors while preserving HS2 totals and active HS6 counts; it is a conditional benchmark, not complete randomization.

Harmonization

HS6 means a six-digit Harmonized System product code. The key problem is that the same six-digit number is not a timeless product identity: the Harmonized System is revised, codes split, codes merge, and reporters adopt revisions at different times.

Why We Harmonize

For a one-year product Gini, a native HS6 code is a useful label. For a time comparison, it can be misleading. If one old code is split into three new codes, a native-code panel can make a classification edit look like product entry. If several old codes merge, it can make product exit or rising concentration appear mechanically.

The estimand for product-level time results is therefore concentration or growth across economically comparable product families, not across whatever native code labels happened to be active in that year.

Where We Do Not Harmonize

Partner-only concentration does not need a product identity after products have been summed into reporter-partner totals. In that case the ranked objects are partner countries, not products, so a value-preserving product remap leaves the partner totals unchanged.

For partner-only measures, HS6 999999 is included in partner totals because it is still trade with a known partner. For product-dependent measures, 999999 is excluded before aggregation because it means Commodities not specified, not a real product.

Product identity rule used for longitudinal product analysis

product identity = connected component of revision-code nodes linked by official WCO adjacent-revision correlations and unchanged-code links

In plain English, each node is a revision-specific code such as H1:010110 for an HS1996 code or H2:010110 for an HS2002 code. We connect nodes when official WCO Table II correlations say the old and new codes correspond, and we also connect unchanged codes across adjacent revisions. The connected component becomes the harmonized product family.

No trade value is split, duplicated, or model-allocated. We only change the product key used for grouping. If two rows map to the same family inside a reporter-year-flow, their trade values are summed.

Very large connected components are treated as ambiguous rather than collapsed. Those nodes stay revision-specific, so the pipeline does not force broad many-to-many code tangles into one artificial product.

Why not convert everything to the latest HS revision?

A target-revision crosswalk sounds cleaner than it is. HS revisions contain one-to-many and many-to-one changes. Without product-level split weights, assigning an old code to a single new code is arbitrary; splitting its value across new codes would add model-generated trade values. That would be especially dangerous for Gini, top-product, entry, exit, and transition statistics.

Coarser HS4 or HS2 groupings are useful robustness checks, but they answer a different question. They measure concentration across headings or chapters, not across HS6-level product families. Native HS6 counts are kept as a diagnostic because they show how much the raw code universe changes over time.

HS6 code-count diagnostic

Nt = count distinct HS6 codes h with positive trade value in year t and h ≠ 999999

Unit before aggregation: reporter-year-flow-HS6 product value. The any-flow count treats an HS6 code as present if it appears in exports, imports, or both. HS6 999999 means Commodities not specified, so it is excluded before the count.

Reading: the any-flow rd2 universe has 5,034 observed HS6 codes in 1988 and 5,717 in 2025; the maximum is 5,877 in 2007.

World-Relative Product Gini

This export-only measure asks whether a country's harmonized product-family basket is uneven relative to the product composition of world trade, rather than only unequal across its own active native HS6 lines.

Feynman version: think of world trade as the reference recipe. If the world recipe is 10 percent cars and 1 percent coffee, we should not judge both products as equally sized slots just because both have HS codes. This measure asks whether a country follows that world recipe or puts unusually large weight on a few ingredients.

Main website measure

scpt = xcpt / Σpxcpt,   w-c,pt = (Bpt - xcpt) / Σp(Bpt - xcpt)
WorldRelativeGinict = weighted_gini(scpt / w-c,pt, w-c,pt)

Unit: rd2 country-year export basket across harmonized HS6 product families, balanced over 2000-2024. Benchmark: leave-one-out world_broad exports in the same year. Exclusion: HS6 999999 is dropped before product aggregation.

Appendix-only weighted-share measure

AppendixGinict = weighted_gini(scpt, w-c,pt)

This literal weighted-share version answers a narrower question: how unequal are country export shares after giving more attention to world-important products. It is not highlighted on the website because it can remain positive even when a country exactly mirrors the world product basket.

Latest benchmark year: 2024. Median World-Relative Product Gini is 0.830; median appendix weighted-share Gini is 0.901. Balanced-window starting median is 0.835 in 2000.
